# 🐙 말랑상점 (Mallang Store) - 모바일 쇼핑몰 앱

## 프로젝트 개요

'말랑상점'은 Flutter를 활용하여 개발된 모바일 쇼핑몰 애플리케이션입니다. 사용자 친화적인 인터페이스를 통해 상품을 둘러보고, 상세 정보를 확인하며, 장바구니에 담아 구매하는 일련의 쇼핑 경험을 제공합니다. 세상에서 가장 귀여운 말랑해적단의 해산물 쇼핑 어플로, 핵심 기능을 귀엽게 구현하는 데 중점을 두었으며, 깔끔하고 직관적인 디자인을 목표로 합니다.

## 🚀 주요 기능 및 페이지 설명

### 1\. 첫 터치 페이지 (`thumbnail_touch_page.dart`)

앱을 실행했을 때 가장 먼저 보이는 시작 화면입니다.

  * **역할**: 사용자에게 앱 진입을 유도하는 시작점입니다.
  * **특징**:
      * 화면 전체를 덮는 **탭 감지 영역**이 있어, 어느 곳이든 터치하면 다음 화면으로 넘어갑니다.
      * **외부에서 이미지 경로를 받아** 해당 이미지를 화면 중앙에 표시합니다. (예: `thumbnail_final.png` 또는 `time_sale.png`)
      * 하단에 **깜빡이는 "Touch\!" 텍스트**가 나타나 사용자에게 상호작용을 유도합니다. 이 텍스트는 `BlinkingTouch` 위젯으로 구현되어 시각적인 애니메이션 효과를 제공합니다.
  * **개발자**: 공통

### 2\. 상품 목록 페이지 (`item_list.dart`)

상점에서 판매하는 모든 상품을 한눈에 볼 수 있는 메인 페이지입니다.

  * **역할**: 상품들을 그리드 형태로 나열하고, 상품 상세 페이지 및 장바구니 페이지로의 이동을 지원합니다.
  * **특징**:
      * **상단 앱바 타이틀**: 앱바에 'Store'라는 타이틀이 표시됩니다.
      * **상품 목록**: 각 상품은 **카드 형식**으로 깔끔하게 정돈되어 있습니다. 각 카드에는 상품 이미지, 상품 이름, 상품 가격이 명확하게 표시됩니다.
      * **가격 표시 예외 처리**:
          * 가격이 세 자리 수 이상일 경우 \*\*세 자리마다 콤마(,)\*\*가 찍혀 가독성을 높입니다. (예: 10000원 -\> 10,000원)
          * 가격이 0원일 경우 "무료"라고 표시됩니다.
      * **빈 목록 처리**: 등록된 상품이 하나도 없을 경우 화면 중앙에 **'상품이 없습니다.'** 메시지가 출력됩니다.
      * **로고/배너 영역**: 상단에 **라운드 처리된 카드 형태의 배너** 이미지가 표시됩니다. 이 영역을 탭하면 **타임 세일 전단 페이지(`/time_sale`)로 이동**합니다. 이 배너 이미지는 `sale_banner.png`를 사용하며, 카드와 이미지 모두 둥근 모서리가 적용됩니다.
      * **장바구니 아이콘**: 앱바 우측 상단에 장바구니 아이콘이 있어, 터치 시 장바구니 페이지로 이동합니다.
      * **상품 등록 버튼**: 화면 하단에 플로팅 액션 버튼(FAB)이 있어, 새로운 상품을 등록하는 페이지로 이동할 수 있습니다. 등록 후에는 상품 목록이 자동으로 갱신됩니다.
  * **담당**: 임초희 (프로젝트 협업 리더)

### 3\. 상품 상세 페이지 (`item_detail2.dart`)

선택된 상품의 자세한 정보와 구매/장바구니 담기 기능을 제공하는 페이지입니다.

  * **역할**: 사용자가 상품에 대한 충분한 정보를 얻고 구매 결정을 내릴 수 있도록 돕습니다.
  * **특징**:
      * **배경 이미지**: 페이지 배경으로 'detail\_bg2\_noshad.png' 이미지가 깔려 있습니다.
      * **상품 정보**: 상품의 **이미지, 상품명, 상세 설명**이 화면에 표시됩니다.
      * **상품 개수 조절**: 사용자 인터페이스를 통해 구매할 상품의 **개수를 손쉽게 조절**할 수 있습니다 (1개 이상 99개 이하).
      * **연상되는 금액 출력**: 선택된 상품의 개수에 따라 **실시간으로 총 금액이 업데이트**되어 표시됩니다. 이 금액 역시 세 자리마다 콤마가 찍혀 가독성이 좋습니다.
      * **구매하기 버튼**:
          * 버튼을 누르면 구매 확인 \*\*다이얼로그(CupertinoAlertDialog)\*\*가 나타나 사용자의 최종 의사를 확인합니다.
          * 구매 확인 시, 선택된 개수가 1개 이상인지 확인하는 로직이 내부적으로 포함되어 있습니다. (현재 코드는 항상 1개 이상이므로 별도 UI 체크는 없지만, 수량 조절에서 이미 최소 1개로 제한됩니다.)
          * 구매 완료 시 '🎉 감사합니다\! 🎉' 메시지를 담은 다이얼로그가 표시된 후 자동으로 상품 목록 페이지로 돌아갑니다.
      * **장바구니 버튼**:
          * 버튼을 누르면 장바구니 담기 확인 다이얼로그가 나타납s니다.
          * 장바구니 담기 확인 시, 해당 상품과 수량을 `CartProvider`에 추가하고 '장바구니에 담겼습니다\!' 메시지와 함께 장바구니 페이지로 바로 이동할 수 있는 옵션을 제공합니다.
  * **담당**: 정소린

### 4\. 상품 등록 페이지 (`item_register_v2.dart`)

새로운 상품을 상점에 추가할 수 있는 페이지입니다.

  * **역할**: 상점 관리자가 새로운 상품 정보를 입력하고 등록할 수 있는 인터페이스를 제공합니다.
  * **특징**:
      * **앱바 타이틀**: '상품 등록'과 같은 명확한 앱바 타이틀이 포함될 예정입니다.
      * **입력 필드**: 상품의 **이미지, 상품명, 상품 가격, 상품 설명** 등을 입력할 수 있는 필드가 제공됩니다. (현재 제공된 코드는 `item_detail2.dart`와 동일하여 입력 필드 구현이 필요합니다.)
      * **등록하기 버튼**: 모든 상품 정보를 입력한 후 버튼을 누르면, 해당 상품이 `ItemProvider`를 통해 상품 목록에 추가됩니다.
  * **담당**: 김영민

## 🛒 도전 기능: 장바구니 (`item_cart.dart` 및 `cart_provider.dart`)

장바구니 기능은 앱의 핵심 구매 흐름을 완성하는 중요한 도전 기능으로 구현되었습니다.

  * **`item_cart.dart`**: 사용자가 장바구니에 담은 상품 목록을 볼 수 있는 UI를 제공합니다.
      * 각 상품별로 수량 조절 (`Icons.add_circle`, `Icons.remove_circle`) 및 삭제 (`Icons.clear`) 기능을 제공합니다.
      * 장바구니에 담긴 모든 상품의 총 가격을 실시간으로 표시합니다.
      * 총 가격 확인 후 '구매하기' 버튼을 통해 전체 장바구니 상품을 구매 완료 처리할 수 있습니다.
  * **`cart_provider.dart`**: 장바구니 상태를 중앙에서 관리하는 `ChangeNotifier`입니다.
      * `addToCart()`: 상품을 장바구니에 추가합니다. 이미 있는 상품이면 수량을 증가시킵니다.
      * `removeFromCart()`: 특정 상품을 장바구니에서 제거합니다.
      * `updateQuantity()`: 장바구니 내 특정 상품의 수량을 업데이트합니다.
      * `totalPrice` Getter: 현재 장바구니에 있는 모든 상품의 총 가격을 계산하여 반환합니다.
      * `clearCart()`: 장바구니의 모든 상품을 비웁니다.
  * **담당**: 이상록

## 📋 Dart 코딩 컨벤션

프로젝트의 코드 일관성과 가독성을 높이기 위해 다음 컨벤션을 준수합니다.

1.  **파일명**: `snake_case` 사용 (소문자 + 언더스코어)
      * 예: `item_list.dart`
2.  **클래스명/위젯명**: `PascalCase` (파스칼 케이스) 사용
      * 예: `class ItemList {}`
3.  **함수명 / 변수명**: `camelCase` (카멜 케이스) 사용
      * 예: `int itemCount = 0;`

## 📝 Git Commit 컨벤션

커밋 메시지의 일관성을 유지하고 변경 이력을 명확하게 추적하기 위해 다음 규칙을 따릅니다.

| 항목       | 설명                                                                                                                                              |
| :--------- | :------------------------------------------------------------------------------------------------------------------------------------------------ |
| **제목** | - **최대 50자**\<br\>- **첫 글자 대문자**\<br\>- **명령문으로 작성** (`Fix`, `Add` 등)\<br\>- **마침표(`.`) 금지**\<br\>- **본문과 한 줄 띄움** |
| **본문** | - **무엇을, 왜 했는지** 설명 (어떻게 했는지보다 무엇을, 왜 했는지에 초점)\<br\>- 여러 줄은 `-` (하이픈)으로 구분                                                             |
| **꼬릿말** | - 선택 사항 (optional)\<br\>- 관련 이슈가 있을 경우에만 작성\<br\>- 형식:\<br\>  `Fixes: #이슈번호` \<br\>  `Resolves: #이슈번호` \<br\>  `Ref: #이슈번호` \<br\>  `Related to: #이슈번호` |

### **Commit Type**

| 타입         | 설명                                      |
| :----------- | :---------------------------------------- |
| `feat`       | **기능**: 새로운 기능 추가                    |
| `fix`        | **버그**: 버그 수정                         |
| `refactor`   | **리팩토링**: 기능 변화 없이 코드 구조 개선     |
| `design`     | **디자인**: CSS, UI 변경 등 시각적 요소 수정  |
| `comment`    | **주석**: 주석 추가, 수정, 삭제               |
| `style`      | **스타일**: 코드 포맷, 세미콜론, 공백 등 (로직 변경 없음) |
| `docs`       | **문서**: 문서 추가, 수정, 삭제 (예: `README`)  |
| `test`       | **테스트**: 테스트 코드 추가, 수정, 삭제       |
| `chore`      | **기타**: 빌드, 패키지, 설정 등 기타 개발 작업 |
| `init`       | **초기화**: 프로젝트 초기 생성 작업             |
| `rename`     | **이름변경**: 파일/폴더 이름 변경 또는 이동     |
| `remove`     | **삭제**: 파일 삭제만 한 경우                 |
